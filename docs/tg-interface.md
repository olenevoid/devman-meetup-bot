# Telegram-интерфейс (attendee-часть)

Документ описывает экраны бота для гостей и спикеров. Флоу организатора
здесь не описаны — они живут вне бота.

## Конвенции

### Кнопка «Меню»

На **любой** клавиатуре, **кроме главной**, присутствует кнопка:

```
[ Меню ]
```

Она возвращает пользователя к главному меню (edit-in-place, см. ниже).
На самом главном меню этой кнопки нет — пользователь уже там.

### Навигация

- **Edit-in-place:** бот редактирует одно и то же сообщение при переходах.
Чат остаётся чистым, история не плодится.
- **Главное меню** — единственное закреплённое сообщение, точка возврата.
- **Callback-data:** `<flow>:<action>:<id>`, например `ask:pick:12`,
`net:next:0`, `spk:page:2`.
- **Inline-кнопки только**, без reply-клавиатуры.

### Роли

У пользователя может быть **несколько ролей одновременно** (модель
`UserRole` допускает `guest + speaker`, `speaker + organizer` и т.п.).
Состав кнопок главного меню **вычисляется из множества ролей** —
пользователь видит ровно те кнопки, которые доступны хоть одной из его
ролей.

| Роль | Доступные блоки |
|---|---|
| `GUEST` | Программа, Спросить, Нетворкинг, Донат |
| `SPEAKER` | + `🎤 Мой доклад (N)` — старт/стоп доклада + вопросы (live-счётчик неотвеченных) |
| `ORGANIZER` | (флоу организатора вне бота — кнопок не добавляет) |

Дублирования кнопок нет: каждая добавляется максимум один раз, даже если
подходит под несколько ролей. Если у пользователя нет ни одной роли —
ему доступны только гостевые блоки (поведение по умолчанию).

## Навигационная карта

Полная схема навигации — в [`tg-navigation.drawio`](./tg-navigation.drawio)
(открывается в [draw.io](https://app.diagrams.net/) или VS Code с плагином
*Draw.io Integration*).

Условные обозначения на схеме:

- **Синий** — Главное меню.
- **Зелёные** — экраны, доступные роли `GUEST` (программа, спросить,
  нетворкинг, донат).
- **Оранжевые** — экраны, доступные только роли `SPEAKER`.
- **Красные** — терминальные/ошибочные ветки (нет активного доклада,
  пустой пул нетворкинга).
- **Сплошные стрелки** — переходы вперёд (с подписями действий).
- Кнопка `[ Меню ]` на схеме не рисуется: она есть на **любом** экране,
  кроме Главного меню, и возвращает в него (см. раздел «Конвенции»).

## Экраны

### 1. Главное меню (Home)

Единственный экран **без** кнопки `Меню`. Состав кнопок собирается из
множества ролей пользователя (см. раздел «Роли»).

**Только гость (`GUEST`):**
```
╭──────────────────────────────────╮
│ 🐍 PythonMeetup                  │
│ Привет! Помогу: программа,       │
│ вопрос спикеру, найти собесед-   │
│ ника, поддержать донатом.        │
╰──────────────────────────────────╯
[ 📅 Программа  ][ ❓ Спросить   ]
[ 🤝 Нетворкинг ][ 💰 Донат      ]
```

**Гость + спикер (`GUEST + SPEAKER`)** — добавляется строка с входом в
кабинет спикера (старт/стоп доклада + вопросы, live-счётчик неотвеченных):
```
[ 📅 Программа  ][ ❓ Спросить   ]
[ 🤝 Нетворкинг ][ 💰 Донат      ]
[ 🎤 Мой доклад (2)             ]
```

**Организатор (`ORGANIZER`)** — флоу организатора живёт вне бота, поэтому
ему показывается тот же набор, что и гостю (гостевые блоки доступны по
умолчанию).

### 3. Спросить спикера (ConversationHandler, 2 шага)

Спикер не выбирается — вопрос всегда уходит **активному** спикеру
(определяется по `Talk.state=active`). Имя активного спикера показывается
в тексте сообщения.

**Шаг 1 — ввод вопроса** (свободный текст одним сообщением):
```
╭──────────────────────────────────╮
│ ❓ Активный спикер: @ivanov       │
│ Доклад: «Async Python»           │
│                                  │
│ Напиши вопрос одним сообщением.  │
╰──────────────────────────────────╯
[ ✖️ Отмена ][ Меню ]
```

**Шаг 2 — подтверждение:**
```
╭──────────────────────────────────╮
│ ✅ Вопрос отправлен @ivanov.     │
│    Он увидит его в разделе       │
│    «Мои вопросы».                │
╰──────────────────────────────────╯
[ ❓ Ещё вопрос ][ Меню ]
```

**Нет активного доклада:**
```
╭──────────────────────────────────╮
│ ❌ Сейчас никто не выступает.    │
│    Попробуй, когда начнётся      │
│    следующий доклад.             │
╰──────────────────────────────────╯
[ Меню ]
```

**Поведение «во время доклада»:**
- Вопрос уходит спикеру с `Talk.state=active`.
- Если доклад идёт дольше расписания — остаётся активным, вопрос уходит
  текущему спикеру, а не следующему (блокирует переключение, пока
  `Talk.state` не сменится).

### 4. Нетворкинг

**4a. Вступление-объяснение** (снимает страх «как это работает»):
```
╭──────────────────────────────────╮
│ 🤝 Нетворкинг                    │
│ Заполни карточку о себе — бот    │
│ предложит анкеты других гостей.  │
│ Контакт отдаётся только когда    │
│ ты сам откроешь его.             │
╰──────────────────────────────────╯
[ ✍️ Заполнить анкету ]
[ Меню ]
```

**4b. Анкета** (ConversationHandler: `bio → stack → contact`).
На каждом шаге — возможность отмены и возврата в меню:
```
╭──────────────────────────────────╮
│ Шаг 1/3: расскажи о себе         │
│ (короткое bio одним сообщением)  │
╰──────────────────────────────────╯
[ ✖️ Отмена ][ Меню ]
```

**4c. Матч — по одной анкете за раз:**
```
╭──────────────────────────────────╮
│ 👤 Алексей · Python/Django       │
│ «Бэкенд в финтехе, ищу коллаб…»  │
│ Контакт: @alexey                 │
╰──────────────────────────────────╯
[ 💬 Написать ][ ⏭ Следующий ]
[ ✖️ Выйти ][ Меню ]
```

`⏭ Следующий` показывает случайную ещё не просмотренную анкету.
`💬 Написать` открывает контакт; анкета остаётся доступной.

**4d. Пустой пул** (анкет ещё нет):
```
╭──────────────────────────────────╮
│ Пока анкет нет. Сообщу, как      │
│ кто-то заполнит карточку.        │
╰──────────────────────────────────╯
[ Меню ]
```

Когда появляется новая анкета — бот присылает уведомление со ссылкой
на продолжение нетворкинга.

### 5. Донат (YooKassa)

```
╭──────────────────────────────────╮
│ 💰 Поддержать митап              │
│ Помоги покрыть пиццу и зал.      │
╰──────────────────────────────────╯
[ 100 ₽ ][ 300 ₽ ][ 500 ₽ ]
[ ✍️ Другая сумма ]
[ Меню ]
```

Выбор суммы → YooKassa invoice → после `pre_checkout` и успешного платежа
создаётся запись `Donation` (`status=paid`, `paid_at=now`). Подтверждение:

```
╭──────────────────────────────────╮
│ 💚 Спасибо! 300 ₽ получены.      │
╰──────────────────────────────────╯
[ Меню ]
```

### 6. Кабинет спикера (роль `SPEAKER`)

Спикер **управляет своим докладом** (старт/стоп) и видит вопросы к нему.
Вопросы **не форвардятся** в ЛС — спикер заходит сам.

Состояние интерфейса зависит от `Talk.state` (см. ниже). Предполагается
**один доклад на спикера** в текущем мероприятии; если их несколько —
потребуется выбор доклада (не реализовано в MVP).

#### 6.1 Машина состояний доклада (`Talk.state`)

Используется существующее поле `Talk.state` (`CharField(max_length=20)`).
Три значения:

| `state` | Смысл | Переходы |
|---|---|---|
| `planned` | Запланирован (по умолчанию) | → `active` по кнопке «Начать» |
| `active` | Идёт прямо сейчас | → `finished` по кнопке «Завершить» |
| `finished` | Завершён | терминальное, **возврат невозможен** |

Инвариант: **активный доклад может быть только один** (глобально). Кнопка
«Начать» проверяет отсутствие другого `Talk.state=active` и блокируется,
если кто-то уже выступает. Проверка на уровне приложения; для надёжности
можно добавить partial unique constraint в БД.

Гости могут задавать вопросы **только** когда есть `Talk.state=active`
(см. раздел 3 — иначе гость видит экран «Нет активного доклада»).

#### 6.2 Верхняя панель и список вопросов — по состояниям

**`planned`, другого активного нет** — кнопка старта, вопросов ещё нет:
```
╭──────────────────────────────────╮
│ 🎤 Мой доклад: «Async Python»    │
│ Статус: запланирован             │
│ Вопросов пока нет — начни доклад │
╰──────────────────────────────────╯
[ ▶️ Начать доклад ]
[ Меню ]
```

**`active` (этот доклад)** — кнопка стопа, под ней вопросы (пагинация 5):
```
╭──────────────────────────────────╮
│ 🎤 «Async Python» ● идёт         │
│ Неотвечено: 2 · стр. 1/3         │
╰──────────────────────────────────╯
[ ⏹ Завершить доклад ]
[ «Какой runtime лучше для I…» ✉️ ]
[ «Про GIL можно подробнее?»  ✉️ ]
[ «Где взять слайды?»         ✉️ ]
[ «Сравнение asyncio/trio…»   ✉️ ]
[ «Когда релиз 3.13?»         ✉️ ]
[ ◀️ Назад ][ 1/3 ][ Вперёд ▶️ ]
[ Меню ]
```

**`planned`, но активен чужой доклад** — старт заблокирован:
```
╭──────────────────────────────────╮
│ 🎤 Мой доклад: «Async Python»    │
│ Сейчас выступает @petrov.        │
│ Начать можно после завершения.   │
╰──────────────────────────────────╯
[ Меню ]
```

**`finished`** — кнопок управления нет, вопросы остаются доступны для
ответа:
```
╭──────────────────────────────────╮
│ 🎤 «Async Python» ✓ завершён     │
│ Неотвечено: 1                    │
╰──────────────────────────────────╯
[ «Где взять слайды?» ✉️ ]
[ ⬅️ К списку ]
[ Меню ]
```

#### 6.3 Деталь вопроса (одинакова для `active`/`finished`):
```
╭──────────────────────────────────╮
│ ✉️ от @x · 19:04                 │
│                                  │
│ Какой async runtime лучше для    │
│ I/O-heavy нагрузок — asyncio     │
│ или trio?                        │
╰──────────────────────────────────╯
[ 💬 Написать @x ]
[ ✅ Отмечено отвеченным ]
[ ⬅️ К списку ][ Меню ]
```

### Правила для кабинета спикера

- **Кнопка Start/End отражает `Talk.state`:** `planned` → `▶️ Начать`,
  `active` → `⏹ Завершить`, `finished` → кнопки управления нет.
- **Только один `active`:** «Начать» блокируется, если активен чужой
  доклад (показывается имя выступающего).
- **`finished` необратим:** повторный старт невозможен (нет перехода из
  `finished` — req 5).
- **Вопросы видны** при `active` и `finished`; при `planned` их ещё нет
  (гости могут спрашивать только когда доклад идёт).
- **Метка кнопки вопроса:** ~30 символов + `…`, полный текст — в детали
  (лимит inline-кнопки 64 байта).
- **Сортировка:** новые сверху; отвеченные помечаются.
- **`✅ Отмечено отвеченным`** ставит `Question.is_answered=True` и
  обновляет live-счётчик.
- **Live-счётчик** на главной (`🎤 Мой доклад (N)`) — число неотвеченных
  вопросов по докладам спикера; Home-сообщение пере-редактируется.

## Маппинг на модели

| Экран / действие | Модели |
|---|---|
| Главное меню | `TelegramProfile`, `UserRole` |
| Программа | `Event` (`is_current`), `Talk` |
| Спросить спикера | `Talk` (по `state=active`), `Question` |
| Нетворкинг | `NetworkingProfile` (`is_published`) |
| Донат | `Donation` (YooKassa `yookassa_payment_id`) |
| Кабинет спикера | `Talk` (`state`: planned→active→finished), `Question` (`is_answered`) |
